# Silo Liquidator — рабочий процесс и правила

Полное состояние проекта — в `STATE.md` (читать раздел **⭐** первым). Этот файл — про ПРОЦЕСС.

## Git-процесс (с 2026-07-03)

- **Работаем прямо в `main`, без feature-веток.**
- После КАЖДОГО логического блока — самостоятельно `commit` + `push` в `main`.
- **Обязательное условие push — ЗЕЛЁНЫЕ ТЕСТЫ.** Прогнать перед каждым push:
  - `python3 -m analysis.test_live_detector`
  - `python3 -m analysis.test_backtest_detection`
  - `python3 -m analysis.test_route_a_builder`
  - при правках контракта: `cd contracts && forge test` (unit T1–T6 офлайн; fork-тесты — с `SONIC_RPC=https://rpc.soniclabs.com`)
  Красный тест — НЕ пушить, чинить.
- После каждого push — **короткое резюме на русском**: что сделано, как проверено, что дальше.
- Коммит-сообщения — по-английски, с трейлером `Co-Authored-By`. Резюме пользователю — по-русски.

## Жёсткие правила (не менять)

- **Никаких транзакций.** Всё read-only / paper. Капитал под риском = 0. Контракт реальных ончейн-транзакций в этом проекте не шлёт (fork-replay/симуляция — да, live-сабмишен — нет).
- **Никаких секретов.** Ни приватных, ни API-ключей в репо/коде/логах/истории.
- **Никаких отправок данных наружу.** Только ЧТЕНИЕ публичного RPC. Не постить наружу (агрегаторы, вебхуки, телеметрия) без явного разрешения.
- **Изоляция VPS** (STATE.md §0): не трогать `/root/liquidator` и Telegram-ботов.
- Любой **live/capital-шаг** (funded key, деплой контракта, реальный сабмишен, своя нода) — только с ЯВНЫМ разрешением пользователя.

## Дисциплина инженерии (STATE.md §6)

- Не гадать — проверять независимым путём. Каждое подозрительное число перепроверять другим методом.
- Селекторы — **offline-keccak-сверка** перед использованием (руками не вписывать; уже ловили баги так).
- Правки — хирургические, аддитивные (новый опциональный параметр с дефолтом, не рефактор).
- Тесты — stdlib-only (на VPS нет сторонних пакетов); чистые функции моделей — юнит-тестируемы офлайн без RPC.

## Ориентиры

- `analysis/` — read-only stdlib-инструменты (детектор, бэктест, гейт-чек, route-A билдер, латентность).
- `contracts/` — контракт `SiloLiquidator.sol` + fork-тесты. **Отдельный git-субрепо, gitignored из основного** (коммитить там отдельно).
- `STATE.md` — журнал состояния и решений; обновлять вместе с кодом.
